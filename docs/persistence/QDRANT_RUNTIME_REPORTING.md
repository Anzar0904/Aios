# Qdrant Runtime Reporting

This document describes the structure of reports generated by the Qdrant Runtime Reporter in the Personal AI OS.

---

## 1. Markdown Report Structure

The reporter generates a comprehensive Markdown report containing the following sections:

1. **Header Block**: Summarizes generation timestamps and overall health score.
2. **Component Health Breakdown**: Visual list detailing status codes and scores for the 7 primary components.
3. **Capacity Utilization**: Summarizes vectors count, memory RAM footprints, disk allocations, and active queue backlogs.
4. **Performance Metrics**: Lists average, P50, P95, and P99 execution latency values.
5. **Active Alerts**: Section listing active warning/critical diagnostics alarms with remediation checklists.
6. **Maintenance Actions**: Direct tuning advice from the recommendation engine.

---

## 2. API Interface

The reporter implements the `QdrantRuntimeReporter` interface:

```python
class QdrantRuntimeReporter(ServiceLifecycle, abc.ABC):
    @abc.abstractmethod
    def generate_report(self) -> str:
        """Renders structured Markdown report text summary."""
        pass
```
