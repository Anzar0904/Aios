# Production Deployment Checklist

> Pre-deployment verification checklist for production environments.

## Infrastructure

- [ ] Application servers provisioned (4 CPU, 16 GB RAM minimum per instance)
- [ ] PostgreSQL database provisioned and accessible
- [ ] Redis instance provisioned and accessible
- [ ] Qdrant vector database provisioned and accessible
- [ ] Load balancer configured (if horizontal scaling)
- [ ] SSL/TLS certificates installed and valid (check expiry)
- [ ] DNS records configured and propagated
- [ ] Network firewall rules configured

## Security

- [ ] All secrets stored in a secrets manager (not in code or unencrypted files)
- [ ] POSTGRES_PASSWORD is strong and unique (rotated regularly)
- [ ] REDIS_PASSWORD configured (and REDIS_TLS=true)
- [ ] QDRANT_API_KEY configured for cloud/secured deployments
- [ ] All AI provider API keys configured (OPENROUTER_API_KEY, etc.)
- [ ] N8N_API_KEY or credentials configured (if using n8n)
- [ ] Firewall rules: only necessary ports open
- [ ] SSH access restricted to VPN/bastion host
- [ ] POSTGRES_SSLMODE=require (enforce SSL on database connections)
- [ ] Production .env reviewed for accidental test values

## Configuration

- [ ] All required environment variables set (see [configuration.md](configuration.md))
- [ ] config/config.toml reviewed and validated
- [ ] AIOS_LOG_LEVEL=INFO or WARNING (not DEBUG)
- [ ] POSTGRES_CONNECT_TIMEOUT set appropriately
- [ ] REDIS_TIMEOUT configured for network conditions
- [ ] QDRANT_DEFAULT_DIMENSIONS matches embedding model (default: 1536)
- [ ] OmniRoute provider priorities configured in config.toml
- [ ] N8N_SERVER_URL configured (if using n8n)

## Database

- [ ] PostgreSQL migrations run successfully (`python -m aios.migrations`)
- [ ] PostgreSQL backup configured and tested
- [ ] PostgreSQL connection pool size tuned
- [ ] PostgreSQL indexes created and optimized
- [ ] Qdrant collections initialized (auto-created on first run)
- [ ] Qdrant snapshots/backup configured
- [ ] Redis maxmemory policy set (allkeys-lru recommended)

## Application

- [ ] Application code at correct release tag/commit
- [ ] All tests passing (`pytest core/tests/`)
- [ ] Python dependencies up to date and pinned
- [ ] Static analysis / linting passes
- [ ] Application starts without errors on staging environment
- [ ] Health check endpoints responding
- [ ] All service integrations verified (PostgreSQL, Redis, Qdrant, n8n)

## Monitoring & Observability

- [ ] Prometheus deployed and scraping AIOS, postgres-exporter, redis-exporter
- [ ] Grafana dashboards imported and displaying data
- [ ] Alertmanager configured with on-call notification channels
- [ ] Alert rules configured for all thresholds in [monitoring.md](monitoring.md)
- [ ] Loki log aggregation configured
- [ ] Log retention policy configured (minimum 30 days)

## Backup & Disaster Recovery

- [ ] PostgreSQL daily backup cron configured
- [ ] Qdrant daily snapshot automation configured
- [ ] Backup restoration tested on a staging environment
- [ ] Backup retention policy set (30 days for DB, 90 days for config)
- [ ] Disaster recovery runbook documented
- [ ] RTO (2–4 hours) and RPO (24 hours) defined and accepted

## Performance

- [ ] Load testing completed against production-equivalent environment
- [ ] Response time p95 < 500ms under expected load
- [ ] PostgreSQL query performance analyzed (no full-table scans)
- [ ] Redis cache hit rate > 80%
- [ ] Qdrant vector search latency p95 < 100ms

## Documentation

- [ ] [startup.md](startup.md) used to verify startup sequence
- [ ] [troubleshooting.md](troubleshooting.md) available on-call runbook
- [ ] [backup_restore.md](backup_restore.md) referenced in DR plan

- [ ] On-call procedures documented and team notified

## Post-Deployment Verification

- [ ] Smoke tests passing (`./scripts/smoke-tests.sh`)
- [ ] Monitor error rate for first 30 minutes post-deployment
- [ ] Verify all service integrations working (PostgreSQL, Redis, Qdrant)
- [ ] Verify n8n workflows triggering correctly (if configured)
- [ ] OmniRoute model requests succeeding in logs
- [ ] Rollback plan validated and ready if needed
