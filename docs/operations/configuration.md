# Configuration Guide

> Complete configuration reference for all environment variables and settings.

## Configuration Overview

AIOS can be configured via:

1. Environment variables (`.env` file or shell environment)
2. Configuration file (`config/config.toml`)
3. Command-line arguments

**Priority order** (highest wins): CLI args > Environment variables > Config file > Defaults

> **Security Note**: Never commit secrets to version control. Use a secrets manager (AWS Secrets Manager, HashiCorp Vault, etc.) in production.

## Core Configuration

| Parameter | Description | Default | Required |
|-----------|-------------|---------|----------|
| `AIOS_HOME` | Root directory for AIOS installation | — | Yes |
| `AIOS_LOG_LEVEL` | Logging level for the application | INFO | No |

**Example:**

```bash
AIOS_HOME=/opt/aios
AIOS_LOG_LEVEL=DEBUG
```

## n8n (Optional) Configuration

> Sourced from: `core/src/aios/n8n/service.py` — `N8NConfigurationService`

n8n is an optional workflow automation platform. If `N8N_SERVER_URL` is not set, the n8n platform registers in a disconnected state and AIOS runs normally without it.

| Parameter | Description | Default | Required |
|-----------|-------------|---------|----------|
| `N8N_SERVER_URL` | URL of the self-hosted n8n server | http://localhost:5678 | No |
| `N8N_API_KEY` | n8n API key for API authentication (preferred over email/password) | — | No |
| `N8N_EMAIL` | n8n admin email for session-based authentication | — | No |
| `N8N_PASSWORD` | n8n admin password for session-based authentication (store securely) | — | No |
| `N8N_BEARER_TOKEN` | n8n bearer token for stateless authentication | — | No |

**Example (API key authentication — preferred):**

```bash
N8N_SERVER_URL=http://localhost:5678
N8N_API_KEY=<your-n8n-api-key>
```

**Example (session-based authentication — fallback):**

```bash
N8N_SERVER_URL=http://localhost:5678
N8N_EMAIL=admin@example.com
N8N_PASSWORD=<n8n-password>
```

> **Authentication Priority**: `N8N_API_KEY` → `N8N_BEARER_TOKEN` → `N8N_EMAIL` + `N8N_PASSWORD` (session cookie).

## OmniRoute / AI Provider Configuration

> Sourced from: `core/src/aios/services/model_impl.py`, `core/src/aios/providers/registry.py`

OmniRoute is the AIOS model selection and routing layer. It selects the optimal AI provider for each request based on capability, cost, and health. At least one provider API key must be configured.

| Provider | Environment Variable | Default Model | Notes |
|----------|---------------------|--------------|-------|
| OpenRouter | `OPENROUTER_API_KEY` | `qwen/qwen3-coder` | Optional |
| Anthropic / Claude | `ANTHROPIC_API_KEY` | `claude-3-5-sonnet` | Optional |
| Google Gemini | `GEMINI_API_KEY` | `gemini-2.0-flash-exp` | Optional |

**OpenRouter** — `OPENROUTER_API_KEY`

Primary multi-model gateway. Routes requests to 100+ models via a single API. Configured via config/config.toml [llm.openrouter].

- Base URL: `https://openrouter.ai/api/v1`
- Default model: `qwen/qwen3-coder`

**Anthropic / Claude** — `ANTHROPIC_API_KEY`

Direct Anthropic Claude integration. Also accepted via CLAUDE_CODE_API_KEY for Claude Code compatibility.

- Base URL: `https://api.anthropic.com/v1`
- Default model: `claude-3-5-sonnet`

**Google Gemini** — `GEMINI_API_KEY`

Google Gemini provider. Used for embedding generation and as a fallback model provider.

- Base URL: `https://generativelanguage.googleapis.com/v1beta`
- Default model: `gemini-2.0-flash-exp`

**config/config.toml (OmniRoute defaults):**

```toml
[llm]
provider = "openrouter"
default_model = "qwen/qwen3-coder"

[llm.openrouter]
base_url = "https://openrouter.ai/api/v1"
timeout = 30
```

See [Architecture Diagrams — OmniRoute](../diagrams/omniroute.md) and [Generated — Providers](../generated/providers.md) for routing details.

> **Security Note**: Store all API keys securely. Never commit them to version control. Use environment variables or a secrets manager in production.
