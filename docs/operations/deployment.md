# Deployment Guide

> Production deployment procedures and best practices.

## Deployment Architecture

AIOS requires the following services in production:

**Required Services:**

- AIOS Core
- PostgreSQL (port 5432)
- Redis (port 6379)
- Qdrant (port 6333)

**Optional Services:**

- n8n (port 5678)

## Pre-Deployment Checklist

See [production_checklist.md](production_checklist.md) for a complete pre-deployment checklist.

## Deployment Steps

### 1. Provision Infrastructure

Provision cloud resources:

- Application server (4 CPU, 16 GB RAM minimum)
- PostgreSQL database (managed service recommended: AWS RDS, GCP Cloud SQL)
- Redis instance (managed service recommended: AWS ElastiCache, Redis Cloud)
- Qdrant vector database (Qdrant Cloud or self-hosted)
- Load balancer (if horizontal scaling)

### 2. Configure Networking

Set up security groups and firewall rules:

- Allow inbound on application port (typically 8000)
- Allow outbound to PostgreSQL (port 5432), Redis (port 6379), Qdrant (port 6333)
- Restrict admin/management ports (SSH to VPN/bastion only)
- Enable TLS/SSL for all external connections

### 3. Deploy External Services

Deploy in dependency order (see [startup.md](startup.md)):

```bash
# Deploy PostgreSQL, Redis, Qdrant via cloud provider or Docker
docker-compose -f docker-compose.prod.yml up -d postgres redis qdrant
```

### 4. Configure Environment

Set all production environment variables. See [configuration.md](configuration.md).

```bash
# Required for production
POSTGRES_HOST=<production-host>
POSTGRES_DATABASE=aios_production
POSTGRES_USER=aios
POSTGRES_PASSWORD=<rotate-regularly>
POSTGRES_SSLMODE=require
REDIS_HOST=<production-redis-host>
REDIS_PASSWORD=<redis-password>
REDIS_TLS=true
QDRANT_HOST=<production-qdrant-host>
QDRANT_API_KEY=<qdrant-api-key>
QDRANT_HTTPS=true
OPENROUTER_API_KEY=<openrouter-key>
```

### 5. Run Database Migrations

```bash
python -m aios.migrations
```

### 6. Deploy Application

```bash
# Build and deploy AIOS
./deploy.sh production
```

### 7. Verify Deployment

Run smoke tests:

```bash
./scripts/smoke-tests.sh

# Manual verification
aios --version
curl http://localhost:8000/health
```

## Post-Deployment

1. Configure monitoring (see [monitoring.md](monitoring.md))
2. Set up backup automation (see [backup_restore.md](backup_restore.md))
3. Configure alerting rules in Prometheus
4. Test n8n integration if using workflow automation
5. Document any deployment-specific notes in the runbook

## Rollback Procedure

If deployment fails:

1. Stop the new deployment immediately
2. Restore the previous application version
3. Rollback database migrations if schema changed: `python -m aios.migrations --rollback`
4. Verify system health on all service health endpoints
5. Update incident log and perform post-mortem
