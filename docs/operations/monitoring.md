# Monitoring Guide

> Monitoring metrics, alerts, and observability.

## Monitoring Overview

AIOS exposes metrics for monitoring system health, performance, and resource usage. The recommended monitoring stack is Prometheus + Grafana + Alertmanager + Loki.

## Service Metrics

| Metric | Description | Alert Threshold |
|--------|-------------|------------------|
| API Response Time | Average response time for API calls | > 500ms p95 |
| Error Rate | Percentage of requests resulting in errors | > 1% |
| OmniRoute Provider Latency | Per-provider LLM response latency | > 10s p99 |
| OmniRoute Provider Failures | Consecutive failures triggering failover routing | > 3 consecutive failures |
| n8n Workflow Failures | Failed n8n workflow executions per hour | > 5 failures/hour |

## Database Metrics

| Metric | Description | Alert Threshold |
|--------|-------------|------------------|
| PostgreSQL Connection Pool | Number of active database connections | > 80% of max_connections |
| PostgreSQL Query Latency | Average PostgreSQL query execution time | > 200ms p95 |
| Redis Memory Usage | Redis memory consumption | > 80% of maxmemory |
| Redis Eviction Rate | Rate of keys evicted from Redis cache | > 0 evictions/s (investigate maxmemory policy) |
| Qdrant Collection Size | Number of vectors in each Qdrant collection | N/A (monitor for unexpected growth) |
| Qdrant Search Latency | Vector similarity search response time | > 100ms p95 |

## System Metrics

| Metric | Description | Alert Threshold |
|--------|-------------|------------------|
| CPU Usage | System CPU utilization | > 80% |
| Memory Usage | System memory utilization | > 85% |
| Disk Usage | Disk space utilization | > 90% |

## Monitoring Stack

Recommended monitoring tools:

- **Prometheus**: Metrics collection and storage
- **Grafana**: Metrics visualization and dashboards
- **Alertmanager**: Alert routing and notifications
- **Loki**: Log aggregation

## Setting Up Monitoring

### 1. Install Monitoring Stack

```bash
docker-compose -f docker-compose.monitoring.yml up -d
```

### 2. Configure Prometheus

Add AIOS as a scrape target in `prometheus.yml`:

```yaml
scrape_configs:
  - job_name: 'aios'
    static_configs:
      - targets: ['aios:8000']
  - job_name: 'postgres'
    static_configs:
      - targets: ['postgres-exporter:9187']
  - job_name: 'redis'
    static_configs:
      - targets: ['redis-exporter:9121']
```

### 3. Import Grafana Dashboards

Import pre-built dashboards from `monitoring/grafana/dashboards/`.

### 4. Configure Alerts

Configure alerting rules in Prometheus and notification channels in Alertmanager.

## Health Checks

AIOS provides health check endpoints:

| Endpoint | Description |
|----------|-------------|
| `/health` | Overall system health |
| `/health/db` | PostgreSQL database connectivity |
| `/health/redis` | Redis connectivity and memory |
| `/health/qdrant` | Qdrant connectivity and collection status |
| `/health/n8n` | n8n server connectivity (if configured) |

**Quick health check commands:**

```bash
# PostgreSQL
psql -U $POSTGRES_USER -d $POSTGRES_DATABASE -c 'SELECT 1;'

# Redis
redis-cli -h $REDIS_HOST -p $REDIS_PORT ping

# Qdrant
curl http://$QDRANT_HOST:$QDRANT_PORT/health

# n8n (if configured)
curl $N8N_SERVER_URL/healthz
```

## Log Management

Logs are written to:

- **Application logs**: `$AIOS_HOME/logs/aios.log`
- **Error logs**: `$AIOS_HOME/logs/error.log`
- **Access logs**: `$AIOS_HOME/logs/access.log`

Configure log retention and rotation using logrotate or your logging platform.
