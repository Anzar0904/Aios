## Purpose

`docs/engineering/ai_development_standards/` establishes the guidelines for AI-assisted development, ensuring that code generated by LLM providers is secure, efficient, and consistent. It answers:

> **How must AI agents generate, validate, and integrate code in this system?**

Adherence to these standards is mandatory for all AI-driven code alterations in the monorepo.

## Quick Reference — AI Engineering Constraints

These constraints govern all AI coding tasks:

| Constraint | Requirement | Reference Document |
|------------|-------------|--------------------|
| **Linter Checks** | Must pass `ruff check` and `ruff format` before staging | [review_and_validation.md](review_and_validation.md) |
| **Commit Format** | Mandatory Conventional Commit message with co-authorship tags | [review_and_validation.md](review_and_validation.md) |
| **Prompt Location** | All prompts reside under `prompts/` - no hardcoded inline prompts | [prompt_engineering.md](prompt_engineering.md) |
| **Type Safety** | No `Any` types; 100% type hints on public methods | [code_generation_rules.md](code_generation_rules.md) |
| **File Mutation Guard** | Must use `ActionEngine` and cache target file state | [safety_guardrails.md](safety_guardrails.md) |
| **Input Validation** | File paths resolved and checked against workspace containment | [safety_guardrails.md](safety_guardrails.md) |
| **Subagent Loop** | Async delegation with `send_message` - no polling | [ai_agent_workflow.md](ai_agent_workflow.md) |
