## 1. Automated Verification Loop

AI-generated code must pass the following verification loop before being submitted:

```
 [AI Edit] ➔ [ruff Code Format] ➔ [ruff Code Linter] ➔ [mypy Type Audit] ➔ [pytest Execution]
```

### Validation Gates
1. **Linter Sweeps**: Run `ruff check ./core` and `ruff format --check ./core` to ensure styling and formatting compliance.
2. **Type Verification**: Run `mypy core/src/aios/` to check type hint correctness and resolve typing violations.
3. **Pytest Run**: Run the unit and integration test suite to verify that all tests pass.

---

## 2. Test Coverage Audits

* **85% Coverage Threshold**: New or modified files must maintain at least **85%** statement and branch coverage.
* **Coverage Sweep Command**: Run the coverage audit before staging changes:
  
  ```bash
  PYTHONPATH=. pytest --cov=core --cov-branch --cov-report=term-missing
  ```

* **No Missing Tests**: If code modifications add branches or methods, corresponding unit tests must be added to verify those paths.

---

## 3. Commit Format Requirements

All commits generated by AI agents must use the **Conventional Commits** format and include co-authorship metadata to distinguish them from human-authored changes:

### Commit Format Schema
```text
[Type]: [Short descriptive summary in imperative mood]

[Optional body detailing implementation decisions and tradeoffs]

Co-authored-by: AI-agent <assistant@personal-ai-os.local>
```

### Allowed Types
* `feat`: Adding a new feature or service capability.
* `fix`: Correcting a bug or regression.
* `docs`: Documentation updates.
* `test`: Adding or modifying tests.
* `refactor`: Restructuring code without changing behavior.
